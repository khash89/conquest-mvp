- **Iterate weekly** - Small improvements every week are better than big launches

- ❌ Planning everything upfront
- ❌ Building in isolation
- ❌ Following rigid 12-week roadmaps
- ❌ Perfecting before shipping
- ❌ Building features nobody asked for

## 🎮 MY APPROACH: AGILE GAME DEVELOPMENT

### The Cycle (Every Week)

```
MONDAY: Read feedback from last week's testers
  ↓
IDENTIFY: What's the #1 complaint or request?
  ↓
TUESDAY-WEDNESDAY: Build 1 improvement with Claude Code
  ↓
THURSDAY: Deploy to GitHub Pages
  ↓
FRIDAY: Send new version to friends
  ↓
WEEKEND: Collect feedback, plan next week
```

**Key rule:** One improvement per week. Not five. One.

## 📋 MY RULES

### SHIP EARLY
- Week 1: MVP is live (even if 50% done)
- Week 2+: New version every Friday
- Don't wait for perfection

### TEST WITH FRIENDS
- Recruit 10 beta testers (friends)
- They test every Friday
- They tell me what to build next
- Their feedback is the roadmap

### ITERATE BASED ON FEEDBACK
- I don't decide what's fun - they do
- If 3+ friends say something → I fix it
- If nobody mentions it → Don't build it
- Players are always right

### DEPLOY EVERY WEEK
- Every Friday: New version live
- Every Friday: Friends test it
- Visible progress every single week

### MEASURE PROGRESS
- Fun score (1-10) - Do friends want to play?
- Playtime - How long do they play?
- Top complaint - What bothers them most?
- Top request - What do they want?

### FOCUS ON ONE GAME
- Build ONE game until it's good
- Get it to 8+/10 fun score
- Then add monetization
- Then build next game
- Not multiple games at once

## 🚀 MY DEVELOPMENT STAGES

### Stage 1: MVP (Week 1)
- **What:** Simplest possible game (playable in 5 min)
- **How:** Ask Claude Code to strip down to essentials
- **Deploy:** Friday to GitHub Pages
- **Test:** Friends play
- **Success:** Game doesn't crash, players understand rules

### Stage 2: Core Loop (Weeks 2-4)
- **What:** Make core gameplay addictive
- **How:** Iterate based on feedback
- **Deploy:** Every Friday
- **Test:** Same 10 friends
- **Success:** Friends want to play again, fun score 8+/10

### Stage 3: Depth (Weeks 5-8)
- **What:** Add variety, progression, replayability
- **How:** Add features players requested
- **Deploy:** Every Friday
- **Test:** Growing number of friends
- **Success:** Playtime increases, players reach end

### Stage 4: Polish (Weeks 9-10)
- **What:** Make it look/feel professional
- **How:** Graphics, sound, UI improvements
- **Deploy:** Every Friday
- **Test:** Public testing (post to Reddit)
- **Success:** No more bugs, feels complete

### Stage 5: Monetization (Week 11)
- **What:** Add ads, in-app purchases, or cosmetics
- **How:** Simple integration, not invasive
- **Deploy:** Friday
- **Test:** Friends willing to spend?
- **Success:** First revenue earned

### Stage 6: Launch (Week 12+)
- **What:** Public marketing and release
- **How:** Post to Reddit, Itch.io, Discord, Twitter
- **Deploy:** Live on all platforms
- **Test:** Real players testing
- **Success:** 100+ players, sustainable income

## 🚦 DECISION FRAMEWORK

Whenever I have a question, I ask:

**Q1: Did friends ask for this?**
- Yes → High priority
- No → Low priority

**Q2: Does it make core game more fun?**
- Yes → Do it
- No → Skip it

**Q3: Can I build it in <3 hours?**
- Yes → Do it this week
- No → Break into smaller pieces

**Q4: Can I deploy it by Friday?**
- Yes → Commit
- No → Don't start

If answer to Q3 or Q4 is No → Don't do it.
